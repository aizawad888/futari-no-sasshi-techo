# README

## サービス概要
<!-- どんなサービスなのかを３行で説明してください。 -->
毎日の買い物において、生鮮食品の適正価格を簡単に確認できます。
地域ごと価格差や季節性も取り入れ、賢く買い物を行うためのサポートを提供します。
また、安価な食材から献立を自動で提案し、効率的に節約生活を支援します。


## このサービスへの思い・作りたい理由
<!-- このサービスの題材となるものに関してのエピソードがあれば詳しく教えてください。
このサービスを思いつくにあたって元となる思いがあれば詳しく教えてください。 -->
ひとり暮らしを始めた頃、買い物に慣れず野菜の適正価格も知らないまま買い物をしていました。
なるべく食費にお金をかけず節約したいと考えていたのにも関わらず高い商品ばかり買っていたと思います。
さらにスーパーの生鮮商品は毎日と言っていいほど価格が変動します。
手に取ったときに果たしてこの食材は「高い」のか？「買い」なのか？そして、その食材から作れる料理は何なのか。
簡単に判断してくれるアプリがあれば、時短かつ節約につながると思ったからです。

### ターゲットユーザー
20～40代 男女
- 買い物に慣れていない（適正価格を肌感でわかっていない）人
- 週に１度以上買い物に行く
- 節約志向
- なるべく料理や自炊をする
- 安いものを買って、材料からレシピを考える派の人
- コスパの悪いものを買うと悔しい、割高なものを購入したくない
- 楽して得したい人
- 面倒で煩わしいことは嫌い
- とはいえ毎週同じ献立も嫌


## サービスの利用イメージ
<!-- ユーザーがこのサービスをどのように利用できて、それによってどんな価値を得られるかを簡単に説明してください。 -->
| 課題 | なぜそれが困るのか？ | アプリがどう解決する？ |
|--|--|--|
| スーパーなどに買い物に行き、野菜と値段を見て高いのか安いのか（買うべきかステイか）を判断したい | 1つずつ適正価格を覚えていないので判断材料がなく、結果的に高い商品を購入している可能性がある | その品目の適正価格を表示する |
| 安い材料から何を作るかまで考えたい | 買っても使えなかったり、他に必要なものがあったりして結局出費がかさむため | その食材を使ったレシピを提案 |
いちいち適正価格を調べてられないので品目と値段を見てぱっと判断できるようにしたい | 買い物をしている最中にそんな時間はない | 写真を撮って読み込むか、簡単な入力（ボタン形式など）にする |

## 機能候補
<!-- 現状作ろうと思っている機能、案段階の機能をしっかりと固まっていなくても構わないのでMVPリリース時に作っていたいもの、本リリースまでに作っていたいものをそれぞれ分けて教えてください。 -->
### MVP
- 品目別の適正価格の表示
- 品目からレシピを自動検索し提案する機能

### 本リリース
- 品目別・都道府県別の適正価格の表示
- SNS投稿機能

### 追加機能
- 生鮮食品以外も対応、地域の粒度を上げる
- カメラ機能・写真機能から品目・価格を自動判別
- 位置情報から、更に正確な地域のデータを取得
- 一週間のレシピを保存できる機能（１週間分の買い物をする人）
- 買い物リスト機能
