# README

## サービス概要
日常の小さな変化に気付いてほしいけど、恥ずかしくて言えないことをやさしくサポートするアプリです。
投稿はカテゴリ選択で行い、相手にはヒントだけが届きます。
答えは一定期間後に公開され、リアルでの「ありがとう」を自然に伝えられる仕組みです。

## このサービスへの思い・作りたい理由
私はもともと口下手で、自分の気持ちをうまく伝えるのが得意ではありません。
だからこそ、日常の名もなき家事やちょっとした変化に気付いてもらえたら嬉しいし、逆にパートナーの小さな心遣いにも「ありがとう」と伝えたい気持ちがあります。

「言わなくても察して」「言わなきゃ気づかない」などの小さなすれ違いで会話が減ったりギスギスした関係になってしまう夫婦やカップルを見ました。
お互い元は他人なので、すべてお互いの思い通りに動くなんてことは無理だと思います。

だからといって、タスクやスケジュールなどを何でもかんでも可視化をしてどちらかが責められるような状況も嫌だし、「見てみて！」と露骨にアピールするのは恥ずかしい。
SNSに書くほど大げさなことでもないし、とはいえ、反応が返ってこない壁打ちのような投稿では意味がない…。
さらに、私たち夫婦は共通の趣味も生活スタイルもまったく違うので、普段の会話のきっかけが少ないという悩みもありました。

そんな背景から、
“2人だけの小さな気付きや感謝が自然に伝わるアプリ”
があれば、毎日のコミュニケーションがもっと温かく、心地よくなるのではないかと思い、このサービスを作りたいと考えています。

## ターゲットユーザー
- カップルや夫婦
- お互いの日常の小さな変化に気付いてほしいけど、言いにくいと感じる人
- 記念日など大事な日を忘れずに祝いたい人
- 日々の家事や生活の細かい変化を楽しく共有したい人

## サービスの利用イメージ

| 課題 | なぜそれが困るのか？ | アプリがどう解決する？ |
| --- | --- | --- |
| 言いにくい小さな変化に気付いてほしい | 自分から言うのは恥ずかしい、でも気付いてほしい | カテゴリだけ選ぶ投稿で、相手にはヒントのみ通知 |
| 相手が気付けるか不安 | 気付けなかったら悲しい気持ちになる | 一定期間後に答えが見られる仕組みでモヤモヤを解消 |
| ありがとうを言うタイミングがわからない | 思ったことを口頭で伝えにくい | 答え公開時に固定文で「ありがとう」を伝えるヒントを提示 |
| 記念日を忘れやすい | 生活に追われて大事な日を忘れてしまう | 事前にリマインダーとメッセージ提案を表示 |

## サービスの差別化ポイント・推しポイント
このアプリは「日記」「つぶやき」「タスク管理」ではなく、
“パートナーに小さな変化へ気付いてもらうための、2人専用の気付き共有アプリ”
であることが最大の特徴です。

既存の記録アプリやSNSと大きく異なるのは、
“自分のために残す記録”ではなく、“自然な形で相手とのコミュニケーションを生む仕組み”
に特化している点です。

### 他サービスとの違いが出るポイント

1. 言いづらい小さな変化に特化した投稿設計

カテゴリ＋内容のみのシンプル投稿で、露骨にならず恥ずかしさも少ない。
家事・外見・体調など“気付いてほしい小さなこと”を伝えやすい。


2. 相手に“気付き”を促す通知の工夫

投稿内容そのものではなく、
「外見の変化があったかも」「家の中で何か変わったかも」など
カテゴリに紐づいた固定文だけを相手へ通知

投稿者は内容を見るタイミングを設定可能（一定時間ロック）
→ 「◯◯をやったよ！」という通知だとどうしても露骨なアピールに感じるので一定時間はロック。
→ 答えが出ていない間に「何が変わったんだろう？」というきっかけを与えて、気づく経験を蓄積する。
→ 最終的には答えを出して、気づいてほしい側の気持ちも汲み取る。

こうした「ぼかし通知」は、類似アプリにはない仕組みだと思います。


3. 気付いてほしいレベルの設定

投稿ごとに“察してレベル”を調整でき、
相手側UIで表示優先度が変わる設計（例：赤文字・上部固定など）
→ コミュ力に依存せず、気持ちの強弱を丁寧に伝えられる機能は独自性が高いです。


4. 反応が可視化されない安心設計

もちろん反応があれば嬉しいけれど、逆に反応がないのも凹むので
従来SNSのように「既読」「スルー」「いいね」「お気に入り」などをあえてなくすことで見る側の「義務感」をなくしました。
その代わりに、通知＋投稿内容が“会話のヒント”として機能し、リアルな会話が自然に増えていく設計にします。

「反応を可視化しないUX」はメンタル負荷の少ない差別化要素と考えています。

5. 2人の関係性に特化した“閉じた空間”

公開SNSでも日記でもない、パートナーと2人だけの専用空間
趣味や生活スタイルが違っても、
小さな変化を共有することで会話の種が生まれやすいと思います。

6. 日記でもタスク管理でもない “第三の目的” のアプリ

- 日記 → 思い出を残すもの
- タスク管理 → 役割や責任を整理するもの
- このアプリ → 気付き・感謝・小さな変化を届け、関係をなめらかにするもの
この位置づけが最も大きな差別化ポイントです。

また、「気づき」を蓄積していくことで「こんなことに気づいてほしかったんだ」とか「こんなことをやってくれていたんだ」という
パートナーの新しい一面を見つけることができるようになります。


全体として、
「気付いてほしい」と「ありがとう」を自然に橋渡しする仕組み
に特化している点が、他アプリにはない大きな強みです。


## 機能候補

### MVP
- カテゴリ選択+内容入力による変化投稿
- 相手へのアプリ内通知（ぼんやりしたヒントのみ）
- 一定期間後の答え公開機能
- 固定文による「ありがとう」ヒント表示

### 本リリース
- 内容のテンプレート化（プリセット化）
- 投稿のアーカイブ・検索機能

### 追加機能
- 記念日リマインダー
- 週末に投稿をまとめて振り返る機能
- プッシュ通知

## 使用する技術スタック
- 使用するフレームワーク: Ruby on Rails、JavaScript
- データベース: PostgreSQL
- デプロイ先: Render
- 使用予定のライブラリ: Devise（認証用）、Omniauth（SNS認証）、webpush gem（プッシュ通知）
